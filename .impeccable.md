## Design Context

### Users
B2B professionals who need provable photo evidence at the moment of capture — insurance adjusters, freelance photojournalists, process servers, landlords doing move-in inspections, marketplace sellers. Used in the field, often quickly. This is a workflow tool (closer to DocuSign than Instagram), not a general consumer camera app, even though the UI itself is camera-app-shaped for the hackathon demo.

### Brand Personality
Forensic, precise, evidentiary. The interface should feel like a scientific/forensic instrument — a chain-of-custody log, an oscilloscope, an evidence kit — not a social app. Cold, exact, trustworthy. No playfulness, no warmth-for-its-own-sake.

### Aesthetic Direction
Dark-only. `background #0a0a0b`, `surface #131314`, hairline borders `#27272a` everywhere, primary text `#ffffff`. Monospace-forward: JetBrains Mono for all data/labels/headlines, Inter for prose only. Three verdict colors are the only saturated colors with meaning — `verdict-green #22c55e`, `verdict-amber #f59e0b`, `verdict-grey #71717a` — plus one accent (`#c4b5fd`, lavender/violet) recently introduced for active/interactive states. Corner registration marks on imagery, a 6px colored top edge on verdict cards, uppercase mono labels with wide tracking.

Anti-references: no glassmorphism, no neon glow, no soft rounded consumer-app look, no gradients, no drop shadows for depth.

Currently reads as flat and inconsistent — particularly the Verify tab's verdict screens and the header/tab bar chrome. This polish pass stays inside the existing palette/type system (confirmed — not evolving the direction) but should add real depth, rhythm, and hierarchy so it reads as "engineered instrument," not "flat UI mockup." Scope is flagship-level, not a quick tidy-up — invest real effort in typographic hierarchy, spacing rhythm, and consistency.

### Design Principles
1. Every saturated color means something (verdict tiers + the one accent for interactive/active state) — never purely decorative.
2. Data (hashes, timestamps, addresses, pubkeys) is always monospace, always selectable, truncated only with a clear reason.
3. Depth comes from hairline borders, corner marks, and edge treatments (the 6px verdict-color top edge) — never shadows, blur, or glow.
4. Spacing should feel measured and intentional, like an instrument's grid — not the same padding value repeated everywhere regardless of context.
5. This pass is flagship-quality: real hierarchy and optical alignment, not just consistent margins.
